# Contribution Guidelines

You are encouraged to solve exercises and put your solutions into this repository.
Please find the instructions below on how tou contribute to this repository.

## Step 1: Fork the Repo

1. Click the Fork button (top-right corner of this page).
2. This creates a copy of the repository under your GitHub account.

## Step 2: Clone the Repo

You can use any method to do this really but I prefer [Github Desktop](https://desktop.github.com/download/)

You can find a tutorial for how to use it once you download the application.

## Step 3: Create Your Personal Folder

- Inside of src/ create a folder with your name.
For example:

```markdown
John/
```

- Inside of your folder, create another folder with the current week. For example:

```markdown
John/Week X
```

This is where your code for the week will be placed.

> [!warning]
> Please do not modify or delete other students' files or folders.

## Step 4: Add, Commit, and Push Your Work

You can do this with GitHub Desktop.

> [!note]
> Before you do this make sure that you do a git pull before you start any of your work.
>
> You should also do a git pull before your push your code to GitHub.

## Step 5: Create a Pull Request (PR)

1. Go to your forked repo on GitHub.
2. Click "Contribute: -> "Pull Request"
3. Ensure the base repo is this repository on branch main, not master.
4. Add a short descriptive title.
5. Click "Create Pull Request"

## Wait for a Review

1. I'll review your PR.
2. You may get comments or requests to fix issues.
3. Once approved your work will be merged into the main repository.

> [!note]
> Always pull the latest changes from the main repository before submitting any code.
>
> Commit **only your own** work.
>
> Follow naming conventions.
